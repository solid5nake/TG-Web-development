Om deze vergelijking te kunnen maken binnen de if statement heb je een logical operator nodig. Voor de opdracht is gebruik gemaakt van de logical AND (&&) operator. 

expressie1 && expressie2

logical AND (&&) operator (logical conjunction) voor een set operanden is true alleen wanneer alle operanden truthy zijn. Het wordt typisch gebruikt met Boolean logische values. Wanneer dat zo is geeft het een Boolean terug. De operator && retourneert in feite de waarde van een van de opgegeven operanden, dus als deze operator wordt gebruikt met niet-Booleaanse waarden, retourneert deze een niet-Booleaanse waarde.

de logical AND operator wordt geevalueerd van links naar rechts, geëvalueerd, hij wordt getest op mogelijke short-circuit evaluation.

Om && logical AND operator te testen voor een mogelijke short-circuit evaluation wordt de volgende regel gehanteerd. (some falsy expression) && expr wordt short-circuit evaluated naar de falsy expression; Dat houdt in dat expr niet geevalueerd wordt, vandaar dat eventuele bijwerkingen hiervan geen effect hebben (bijv. als expr een functieaanroep is, vindt de aanroep nooit plaats). 

        Logical AND assignment short-circuits, wat betekent dat x && = y gelijk is aan:
    
x && (x =y);
        En niet gelijk aan het volgende dat altijd een opdracht zou uitvoeren:
    
x = x && y;

short-circuit evaluation Dit betekent dat wanneer JavaScript een OR-expressie evalueert, als de eerste operand waar is, JavaScript kortsluiting heeft en niet eens naar de tweede operand kijkt. In het onderstaande voorbeeld geven de asterisken (****) een waarde aan - het maakt gewoon niet uit wat het is, want JavaScript zal die kant van de logical OR (||) zelfs nooit lezen.

Example

      true || ****
      // true

      let x = 0;
      let y = 1;

      x &&= 0; // 0
      x &&= 1; // 0
      y &&= 1; // 1
      y &&= 0; // 0
